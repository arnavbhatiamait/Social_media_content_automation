# 📸 Instagram Graph API Setup Guide

This guide walks you through setting up the Instagram Graph API to allow the pipeline to post Images, Reels, Stories, and Carousels automatically.

---

## 🔑 Prerequisites

Before starting, ensure you have:
1.  A **Facebook Personal Account**.
2.  An **Instagram Professional Account** (Business or Creator type - Personal accounts do not support API publishing).
3.  A **Facebook Page** (which you must manage).

---

## 🛠️ Step 1: Connect Instagram to your Facebook Page

Instagram Graph API operations are driven through a linked Facebook Page.
1.  Go to your **Facebook Page**.
2.  Navigate to **Settings** > **Linked Accounts** > **Instagram**.
3.  Click **Connect Account** and log into your Instagram Professional Account.
4.  Verify that the connection is active and that your Instagram username appears correctly in Page Settings.

---

## 🌐 Step 2: Create a Meta Developer App

1.  Go to the [Meta for Developers Portal](https://developers.facebook.com/) and register as a developer.
2.  Click **My Apps** > **Create App**.
3.  Select **Other** as the app type, then click Next.
4.  Choose **Business** (or Custom) as the app type to ensure access to Instagram APIs.
5.  Set your App Display Name and link it to your Meta Business Account if applicable, then click **Create App**.
6.  Once created, copy the following from your App Dashboard:
    *   **App ID** (Corresponds to `META_APP_ID` in `.env`)
    *   Navigate to **Settings** > **Basic** to find your **App Secret** (Corresponds to `INSTAGRAM_APP_SECRET` in `.env`).

---

## 🔑 Step 3: Get a User Access Token

To authenticate your requests, you need a User Access Token.
1.  Go to the [Meta Graph API Explorer](https://developers.facebook.com/tools/explorer/).
2.  In the **Meta App** dropdown, select the app you created in Step 2.
3.  Under **User or Page**, choose **Get User Access Token**.
4.  In the **Permissions** section, add the following scopes:
    *   `instagram_basic`
    *   `instagram_content_publish`
    *   `pages_read_engagement`
    *   `pages_show_list`
    *   `business_management`
5.  Click **Generate Access Token** and log in via Facebook to grant permissions.

---

## ⏳ Step 4: Convert to a Long-Lived Token (60 Days)

Tokens generated in the Explorer tool expire in 1-2 hours. You must convert it to a long-lived access token.

Run the following HTTP request in your browser or Postman (replace variables with your values):

```http
GET https://graph.facebook.com/v25.0/oauth/access_token?  
    grant_type=fb_exchange_token&          
    client_id={META_APP_ID}&
    client_secret={INSTAGRAM_APP_SECRET}&
    fb_exchange_token={SHORT_LIVED_TOKEN}
```

**Response:**
The API will return a JSON containing the new `access_token` and an `expires_in` field (~5184000 seconds / 60 days). 
Copy this long-lived token and paste it into your `.env` file as:
```env
INSTAGRAM_ACCESS_TOKEN=EAAT6uaTZA1wIB...
```

> [!TIP]
> For production environments, it is recommended to create a **System User** token inside your Meta Business Manager. System User tokens do not expire.

---

## 🔍 Step 5: Discover your Instagram Account ID

The pipeline needs your **Instagram Business Account ID** to publish content (which is different from your Instagram personal/username ID). We provide automated scripts to find and configure this.

1.  Ensure you have added your `INSTAGRAM_ACCESS_TOKEN` to your `.env` file.
2.  Run the discovery script:
    ```bash
    python scripts/discover_insta_id.py
    ```
    This script queries your Facebook Pages, finds the linked Instagram Business Account ID, and automatically updates the `INSTAGRAM_ACCOUNT_ID` in your `.env` file!

3.  Alternatively, you can run the manual inspection helper:
    ```bash
    python scripts/find_instagram_id.py
    ```
    This prints out the name, ID, and follower count of all linked Instagram accounts for manual pasting:
    ```env
    INSTAGRAM_ACCOUNT_ID=17841420162172489
    ```

---

## 🩺 Step 6: Verify and Troubleshoot

If you run into issues posting reels or images, run the diagnostics tool:
```bash
python scripts/debug_instagram.py
```

This script runs tests to check:
*   Token expiration status.
*   Token owner identity and permissions.
*   Whether the `INSTAGRAM_ACCOUNT_ID` in `.env` is accessible.
*   Which scopes/permissions are actually granted to the token.

### Common Problems & Solutions

1.  **"Media container is not ready / ERROR status"**:
    *   Instagram needs to process media. If a reel fails, ensure the video is in a supported format (H.264 video codec, AAC audio codec, vertical aspect ratio 9:16, $\le$ 60 seconds).
    *   Verify the GCS signed URL generated by the storage pipeline is working and accessible from outside your local network. Instagram's servers *must* be able to download the media from the URL.
2.  **"Permission Error / Unsupported Post Request"**:
    *   Make sure your Instagram account is truly a Business/Professional account.
    *   Re-generate the token and verify that `instagram_content_publish` is listed in the scopes output of `python scripts/debug_instagram.py`.
